# 2026-07-16：小米 14 躺下识别与自动化

## 用户目标

用户希望利用小米 14 类似“识别躺下/注视感知”的能力触发自动化，并要求直接建立真实代码项目。

## 已确认事实与决策

- 不依赖未公开的小米系统私有接口。
- 第一版使用 Android 公开的重力、加速度和陀螺仪传感器。
- 第一版检测的是“手机姿态与稳定性代理条件”，不是直接识别人体躺卧。
- 项目名称暂定为 `sleep-sense`。
- 项目放入总仓库 `paperplane123/GPTchatAllRepo` 的 `projects/sleep-sense/`。
- 后续本对话产生的代码、技术文档和阶段性结论默认继续提交到该仓库。
- 创建 GitHub 仓库由用户完成；仓库内其余开发工作由助手执行。

## MVP 状态机

```text
UPRIGHT -> CANDIDATE -> LYING_STABLE
```

进入稳定状态后，通过自定义 Android 广播向 Tasker、MacroDroid 或后续自研自动化模块发出事件。

## 后续方向

- 增加前台服务和低功耗持续检测。
- 增加时间、环境光、屏幕状态等上下文。
- 使用 CameraX + MediaPipe 估计头部姿态，接近系统级注视感知体验。
