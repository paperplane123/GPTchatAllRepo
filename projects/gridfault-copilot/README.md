# GridFault Copilot

**An explainable fault-diagnosability console for distribution-grid operators.**

GridFault Copilot turns sparse steady-state measurements, topology context, protection signals, and telemetry quality into a ranked fault-section assessment. More importantly, it can say **“the evidence is not sufficient”** and recommend the next measurement or switching check instead of forcing a confident answer.

This repository contains the OpenAI Build Week MVP. It was designed and implemented with Codex/ChatGPT collaboration.

## Why this matters

Most fault-location demos only output a class label. Real distribution-grid operators also need to know:

- whether the current event is diagnosable;
- which evidence supports or contradicts the result;
- how confidence changes when telemetry is missing;
- which next action would reduce ambiguity fastest.

GridFault Copilot therefore exposes a first-class **diagnosability head** alongside section ranking and fault-type estimation.

## MVP features

- interactive radial-feeder topology view;
- three runnable demo cases;
- editable fault resistance, voltage deviation, zero-sequence current, telemetry coverage, and protection state;
- explainable, physics-inspired local scoring baseline;
- ranked section probabilities and confidence;
- diagnosable / ambiguous state;
- recommended next measurement or switching verification;
- optional GPT-5.6 operator brief through the OpenAI Responses API;
- local fallback explanation when no API key is configured;
- zero frontend dependencies.

## Run

Python 3.11+ is recommended.

```bash
cd projects/gridfault-copilot
python server.py
```

Open `http://127.0.0.1:8787`.

The UI works without an API key. To enable the GPT-5.6 operator brief:

```bash
export OPENAI_API_KEY="your_key_here"
export OPENAI_MODEL="gpt-5.6"
python server.py
```

The API key remains server-side. Do not place it in browser JavaScript or commit it to the repository.

## Architecture

```text
Operator controls / demo event
            |
            v
Physics-inspired scoring baseline
  - topology prior
  - zero-sequence evidence
  - voltage deviation
  - protection consistency
  - telemetry coverage
            |
            +--> section ranking
            +--> diagnosability state
            +--> next-best observation
            |
            v
Optional GPT-5.6 operator brief
(OpenAI Responses API, server-side)
```

The baseline is deliberately deterministic and inspectable. It is not presented as a trained production model. A production version would replace or augment it with a graph foundation model trained on simulated and field events while preserving the same evidence and diagnosability interfaces.

## OpenAI integration

`server.py` calls `POST /v1/responses` when `OPENAI_API_KEY` is present. The model receives only the compact diagnostic result and is instructed to produce a concise operator brief without inventing measurements.

Default model: `gpt-5.6`.

## Safety and scope

- This is a decision-support prototype, not an autonomous protection system.
- It must not issue breaker commands or bypass dispatch procedures.
- Demo probabilities are generated by an explainable baseline, not validated field performance.
- Production deployment requires utility data governance, cybersecurity review, model validation, and human authorization.
