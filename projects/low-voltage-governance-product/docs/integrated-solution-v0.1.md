# 低电压治理软硬一体化智能决策产品方案 V0.1

> 状态：阶段性方案，不是最终定稿。真实点位、验收阈值、ETAP版本、措施造价和硬件采购价格，须与通科及专业人员确认后固化。

## 一句话定位

不是做一个“AI造数据”的算法课题，也不是把ETAP界面搬到网页上，而是做一个能够从问题发现走到治理复核的专家型产品。

## 1. 建设目标

产品以低电压问题点为基本对象，将数据、人工经验、ETAP仿真、治理措施、投资比选和工单流程组织成可追溯闭环：

```text
发现问题 → 数据校核 → 成因诊断 → 立即处置 → 工程治理 → 仿真比选 → 人工确认 → 工单实施 → 效果复核
```

核心结果包括：

- 标准化单点治理流程；
- 建立规则、物理仿真和AI协同的计算引擎；
- 形成六页面软件产品和便携式硬件终端；
- 先完成一个真实点闭环，再扩展批量建模、仿真和报告；
- 沉淀数据、接口、措施、报告、验收和硬件模板。

## 2. 产品定位与边界

产品暂定名称为“低电压治理智能决策专家”，面向供电所、配网专责、规划建设、管理人员和专家。

产品负责：

- 数据校核、缺失分类和可信度管理；
- 五类低电压成因诊断；
- 负荷和低电压风险预测；
- ETAP自动建模、场景运行和指标提取；
- 管理措施、工程措施和方案组合；
- 投资、周期、停电影响和复发风险比较；
- 工单、报告和治理后复核。

产品不负责：

- 绕过审批直接下发调控指令；
- 使用AI猜测关键工程参数；
- 用生成数据替代真实拓扑、导线型号、配变容量或物理潮流；
- 将演示结果用于施工和验收。

## 3. 总体架构

| 层级 | 核心能力 |
|---|---|
| 应用层 | 驾驶舱、地图、问题点详情、诊断、方案比选、工单、报告、复核 |
| 决策层 | 措施适用性规则、方案组合、约束校核、经济性排序、人工审批 |
| 分析与仿真层 | 不平衡潮流、时序潮流、场景仿真、灵敏度分析、风险预测 |
| 数据治理层 | 数据接入、时间对齐、质量评分、缺失分类、参数版本和溯源 |
| 感知与硬件层 | 表计/采集终端、RS485/以太网、便携一体机、独立三相采集箱 |

规则负责业务与安全约束，物理仿真负责确定性电气计算，AI负责意图理解、预测、检索和解释，人工专家承担关键参数确认和最终审批责任。

## 4. 数据治理

### 4.1 数据范围

- 资产与拓扑：配变、线路、开关、节点、导线型号、长度、连接关系、分接头；
- 运行量测：三相电压/电流、有功/无功、功率因数、负载率、设备状态；
- 用户与负荷：用户相别、类型、典型曲线、敏感用户、充电负荷和大负荷投切；
- 环境与场景：时间、季节、气温、节假日、分布式电源出力和极端天气；
- 业务数据：投诉、历史治理、工单、造价、施工周期、停电影响和验收结果。

### 4.2 缺失分级

| 等级 | 定义 | 处理原则 |
|---|---|---|
| A 已实测 | 现场或监测系统直接获取 | 直接使用，记录时间、来源和质量标识 |
| B 可从系统获取 | 资产、营销、计量或GIS中存在 | 自动拉取并校验版本和一致性 |
| C 可估计 | 短时负荷断点、典型曲线等非关键数据 | 给出区间、方法、置信度和影响，人工确认 |
| D 关键且不可估计 | 拓扑、导线型号、配变容量等 | 阻止正式计算，生成补采或现场核实任务 |

系统必须区分实测值、系统值、估计值和缺失值。估计参数必须可查看、修改和否决。

## 5. 诊断与模型策略

### 5.1 五类成因

1. 电源侧：上级10kV母线偏低、调压或无功支撑不足；
2. 网架侧：供电半径长、导线截面小、联络薄弱、接点压降；
3. 配变侧：容量不足、分接头不当、重过载、台区边界不合理；
4. 用户与负荷侧：三相不平衡、大负荷投切、集中充电、用户侧问题；
5. 运行管理侧：运行方式、无功管理、档案维护、巡检和处置不到位。

### 5.2 神经网络的正确位置

神经网络可用于短期负荷预测、低电压风险预测、非关键时序插补和相似案例检索。首期不预先锁定CNN、LSTM或GNN，应根据任务、数据和评测指标选型。

神经网络不得用于凭空生成关键拓扑、导线型号、配变容量或正式潮流结果。

## 6. ETAP自动建模和方案仿真

ETAP是后端确定性计算引擎之一。用户看到的是治理前后指标和方案比较，而不是软件操作界面。

### 输入

- 拓扑、线路R/X/C、导线型号和长度；
- 变压器容量、阻抗和分接头；
- 负荷、电源、时间断面和分布式电源工况；
- 管理或工程措施对应的参数变化。

### 输出

- 节点三相电压；
- 支路电流、潮流和损耗；
- 设备负载率；
- 三相不平衡和越限情况；
- 治理效果、约束是否满足、风险和推荐顺序。

自动流程：

```text
读取真实点位 → 构建/更新模型 → 运行基准工况 → 生成候选Scenario → 批量计算 → 读取指标与排序
```

第一阶段先复跑一个真实问题点，比较人工报告和系统结果；随后将点位数据映射为统一模板，扩展批量建模和计算。

## 7. 治理措施库与推荐

### 7.1 管理与工程分层

| 类型 | 目标 | 典型措施 | 复发特征 |
|---|---|---|---|
| 管理措施 | 快速恢复、保供和验证成因 | 调档、换相、转供、无功、错峰、临时储能/发电车 | 可能复发 |
| 工程措施 | 改变设备、线路或网架条件 | 换线、增容、新增布点、网架优化、储能、柔性互联 | 显著降低复发 |

### 7.2 推荐优先级

- 0级：数据与安全校核；
- 1级：低成本管理措施；
- 2级：轻量工程；
- 3级：常规线路、配变和网架工程；
- 4级：储能、柔性互联、光储协同等新型灵活技术，必须证明场景必要性。

方案必须同时比较最低电压、最大负载率、三相不平衡、轻载过电压、损耗、投资、工期、停电影响、用地和复发风险。系统推荐1至3套方案，至少覆盖快速处置、效果与投资平衡、根治优先三类思路。

## 8. 软件产品

六个核心页面：

1. 治理驾驶舱；
2. 问题点详情；
3. 智能诊断；
4. 治理措施；
5. 方案比选；
6. 工单与报告。

关键交互原则：先展示数据完整度，再给诊断；诊断必须有证据和不确定性；措施先管理后工程；方案可调参重算；确认后才能生成工单；治理后回填实测并判断达标或复发。

## 9. 硬件产品与华强北样机

第一台样机采用15.6英寸手提箱式低电压治理智能终端：上盖嵌触摸屏，下层安装计算主机、隔离通信和实体交互面板，真实三相采集放入独立DIN导轨箱。

推荐配置：

- x86高性能迷你主机，32GB DDR5，1TB NVMe，双2.5G网口，Windows 11 Pro；
- 15.6英寸1080P IPS电容触摸屏；
- 隔离型USB-RS485，以太网和Wi-Fi；
- 红黄绿状态灯、诊断/方案/工单按键和旋钮；
- RP2040/ESP32 Modbus三相表计模拟器；
- 可选真实三相导轨表、互感器、保护和端子采集箱；
- 铝合金航空箱、散热、固定、线束和标签。

安全红线：主展示箱内部不放裸露220/380V，不使用无隔离廉价RS485板，不允许样机直接作为现场永久设备；真实市电接线由有资质人员完成。

华强北询价目标：

- 安全演示机：5500至7500元；
- 增加真实三相采集箱：6500至9000元；
- 工业级升级版：12000至18000元。

以上仅为询价控制目标，不是正式报价。

## 10. 标准化与产品化

拟沉淀：

- 低电压问题点数据交换规范；
- 仿真场景与结果接口规范；
- 治理措施分类与推荐规则；
- 单点分析报告模板；
- 便携式智能终端技术要求；
- 治理效果评价与复发判定方法。

推进顺序应为：项目模板 → 企业规范 → 多项目验证 → 团体或行业标准建议。首期不直接声称制定行业标准。

## 11. 真实单点试点与路线图

### 真实单点要求

- 提供真实问题点、原始数据、ETAP工程/模型、人工报告、治理措施和造价；
- 复现治理前电压、负载率、不平衡和主要成因；
- 生成至少一套管理措施和两套工程候选方案；
- 比较效果、投资、周期、停电影响和风险；
- 与人工方案对照并记录差异；
- 形成工单、报告和治理后复核。

### 里程碑

| 时间 | 关键成果 |
|---|---|
| 第1周 | 方案V0.1、措施框架、数据清单、调研问题、可点击原型 |
| 第2周 | 六页面原型、华强北样机设计和演示脚本 |
| 第1个月 | 获取真实点位，完成基准模型和单点初步闭环 |
| 第2个月 | 完成ETAP场景批量运行、方案推荐和报告自动生成 |
| 第3个月 | 完成现场验证、治理后复核、标准模板和试点评估 |

## 12. 阶段性结论

当前最正确的路线不是先训练一个复杂神经网络，而是先做出“真实问题点可进入、数据缺失可暴露、ETAP可复跑、方案可比选、工单可生成、治理后可复核”的产品闭环。

软硬一体机用于承载和展示这一闭环，标准化用于把单点成功复制到更多地区。
