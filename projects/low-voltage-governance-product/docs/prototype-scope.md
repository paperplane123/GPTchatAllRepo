# 两周交互原型范围

> 状态：V0.1，演示数据，不代表现场结论。

## 1. 原型目标

在不接入真实生产系统和 ETAP 的前提下，验证低电压治理产品的完整用户路径：

```text
发现问题 → 查看数据 → 发起诊断 → 理解原因 → 选择措施 → 比选方案 → 生成工单/报告
```

原型重点是用户可见的功能和交互逻辑，不讨论模型训练、数据库部署、接口安全等生产实现细节。

## 2. 页面清单

### 页面 A：治理驾驶舱

必须展示：

- 问题点总数；
- 今日新增；
- 高风险点；
- 反复发生点；
- 待生成工单；
- 区域分布和问题点列表；
- 按风险、区域、状态筛选。

关键交互：点击问题点进入详情。

### 页面 B：问题点详情

必须展示：

- 基础档案；
- 近 24 小时电压曲线；
- 最低电压和发生时间；
- 配变最大负载率；
- 三相不平衡度；
- 受影响用户；
- 数据完整度；
- 缺失参数与来源。

关键交互：点击“开始智能诊断”。

### 页面 C：AI 诊断

必须展示：

- 主要成因排序；
- 每个成因的证据；
- 置信度；
- 不确定数据；
- 对结论影响最大的缺失项；
- 建议补采任务。

关键交互：用户可接受、修订或重新分析。

### 页面 D：治理措施

分为两个标签：

- **立即处置/管理措施**；
- **工程治理/根治措施**。

每项措施展示：

- 适用原因；
- 预计效果；
- 实施时间；
- 成本等级；
- 复发风险；
- 前置条件；
- 是否加入候选方案。

### 页面 E：方案比选

至少展示三套候选方案：

- 低成本快速方案；
- 推荐平衡方案；
- 根治性方案。

比较指标：

- 治理后最低电压；
- 最大负载率；
- 三相不平衡度；
- 投资；
- 工期；
- 停电影响；
- 根治性；
- 结论可信度。

关键交互：选择推荐方案、调整参数、重新计算。

### 页面 F：工单与报告

必须展示：

- 问题描述；
- 诊断结论；
- 确认方案；
- 责任人；
- 完成期限；
- 风险提示；
- 验收指标；
- 报告预览。

关键交互：生成演示工单、打印报告。

## 3. 关键状态

原型必须区分以下状态：

- 待分析；
- 数据不足；
- 已诊断；
- 临时恢复；
- 待工程治理；
- 已生成工单；
- 实施中；
- 待复核；
- 已关闭；
- 问题复发。

## 4. 数据缺失交互

数据缺失时不能只显示“暂无数据”，应展示：

- 缺失字段；
- 是否影响正式计算；
- 系统建议区间；
- 估计依据；
- 置信度；
- “采用建议值”或“创建补采任务”按钮。

原型中的建议值必须标记为“估计”。

## 5. 演示脚本

1. 打开驾驶舱，看到一个高风险问题点；
2. 进入“佛山陈村北一号台区（演示）”；
3. 查看晚高峰电压下降和数据缺失提示；
4. 发起智能诊断；
5. 系统给出“线路压降、配变高负载、B 相负荷集中”三类成因；
6. 查看管理措施，选择“调档 + 三相调整”；
7. 查看工程措施，选择“末端换线”和“新增配变布点”；
8. 进入方案比选，选择推荐平衡方案；
9. 生成工单与报告预览；
10. 强调全部结果是演示数据，真实结论需接入现场数据与仿真引擎。

## 6. 原型验收清单

- [ ] 六个页面均可访问；
- [ ] 页面之间可通过按钮和导航切换；
- [ ] “开始智能诊断”有明确反馈；
- [ ] 管理措施与工程措施严格分开展示；
- [ ] 至少三套方案可比较和选择；
- [ ] 数据缺失、估计值和置信度可见；
- [ ] 可生成演示工单；
- [ ] 可打印报告；
- [ ] 所有演示数据均有醒目标识；
- [ ] 不出现“已完成真实 ETAP 计算”的误导性表述。
