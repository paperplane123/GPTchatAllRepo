# 架构说明

## 目标

不是做一个自动回复机器人，而是把群聊变成结构化事件流：

```text
消息适配器 -> 标准消息协议 -> 分析器 -> 事件结果 -> Web/CLI/工单/日历
```

## 当前模块

- `io.py`：读取 JSON，未来适配企业微信、导出文件和 wx-cli；
- `analyzer.py`：可测试的规则基线；
- `domain.py`：消息、待办、风险和分析结果数据结构；
- `app.py`：本地 HTTP API 与 Web 页面；
- `cli.py`：批处理和自动化入口。

## 接入 wx-cli 的原则

wx-cli 只能作为实验适配器，不应与分析内核耦合。适配器只负责把消息转成统一 JSON，核心系统不依赖具体微信实现。个人微信自动化存在账号限制、客户端兼容和合规风险，生产路线应优先企业微信官方能力。

## 模型演进

规则分析器是可解释基线。后续模型接口应遵循：

1. 默认关闭外发；
2. 支持本地/私有 OpenAI-compatible endpoint；
3. 发送前脱敏；
4. 结果保留原消息引用和置信度；
5. 模型失败时回退到规则分析器。
