# WPS Mac 严格只读阅读加载项

> 状态：**停止推荐**。V0.2.0 已完成代码和静态校验，但在用户当前 WPS for Mac 构建中实测为：自定义选项卡可以显示，所有自定义按钮和直接嵌入的 WPS 阅读命令均不执行。

## 实测结论

这不是单个回调函数或按钮配置错误，而是当前 WPS Mac 构建的加载项事件执行链没有正常工作。继续修改 `main.js`、`ribbon.xml` 或重复覆盖安装，无法解决该构建本身不执行事件的问题。

项目代码保留用于：

- 记录 WPS Mac 加载项兼容性问题；
- 供其他 WPS 版本或后续官方修复后复测；
- 避免把“选项卡能出现”误判为“加载项功能可用”。

## 推荐替代方案

使用独立的 macOS 真只读查看器：

```text
projects/mac-readonly-doc-viewer
```

它不依赖 WPS 加载项接口，使用系统 Quick Look 预览 Word/PDF，界面没有正文编辑、删除、粘贴和保存入口。

## 原设计目标

V0.2.0 原计划：

- 保留 WPS 当前文档页面、图片、表格、公式和排版；
- 进入 WPS 阅读版式；
- 设置 `ReadingLayoutAllowEditing = false`；
- 退出时恢复普通页面视图；
- 不保护、不保存、不修改文档。

上述接口在代码层面存在，但用户当前 WPS 构建没有执行对应加载项事件，因此未达到可用状态。

## 卸载

完全退出 WPS 后，在项目目录执行：

```bash
bash uninstall.sh
```

卸载脚本会移除插件注册记录和历史版本目录。
